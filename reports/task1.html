Incident Prediction from Time-Series Metrics¶

Our task requires implementation of a machine learning model that predicts whether an incident will occur in the next H time steps based on the previous W observations of system metrics.

Problem Formulation¶

Let's define m system metrics at time t as $X_t = (X_t^1, X_t^2, ..., X_t^m)$. For a window of time size W the model input will be:.. $$ [X_{t-W+1}, ..., X_t]. $$ The model predicts whether an incident will occur within the next H time steps: $$ y_t = \left\{ \begin{array}{l} 1 \mbox{ if an inciden occurs in } [t+1, t+H], \\ 0 \mbox{ otherwise. } \end{array}\right. $$ This converts the time-series prediction problem into a supervised classification problem.

Synthetic Dataset¶

We generate a synthetic multivariate time-series dataset representing system monitoring metrics. The dataset contains the following metrics:

  • CPU usage,
  • memory usage,
  • request latency,
  • error rate.

We simulate 10,000 time steps. Most of the time, the system is in a "Normal" state with random noise. However, we manually create "Incidents" and additionaly, to simulate reall system behaviour, for 40 steps before each incident, we program a "drift" where the values start to climb.

Sliding Window Formulation¶

As we previously defined, we need to transform a single long table of numbers into "samples". Each sample consists of a window of size W (the past) and a label $\mathbf{y_t}$ (the future). Let's say $W=30, \ H=20$, our goal is to build the model that will look at the last 30 minutes of data to decide if an incident occures at any point in the next 20 minutes. We flatten the window $[W_{t-W+1}, ..., X_t]$ into a long row of features so a standard ML model can read it:

Model training¶

To solve this task we decided to use a Random Forest Clasifier because it's non-parametric and good with outliers and imbalanced data. When evaluating the model using just Accuracy may semms insufficient due to rarity of incidents, we can get good accuracy just by labeling "No Incident" every time. Instead we should consider Precision and Recall.

from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import classification_report, confusion_matrix

clf = RandomForestClassifier(n_estimators=100, class_weight='balanced', random_state=123)
clf.fit(X_train, y_train)

y_pred_probs = clf.predict_proba(X_test)[:, 1]
alpha = 0.2
y_final_pred = (y_pred_probs >= alpha).astype(int)

print(classification_report(y_test, y_final_pred))

Evaluation of Results¶

Let's look at metrics for Class 1 (Incidents): Precision is a critical metric for alerting systems. It says that our model predicts an incident is about to occur within next H steps correctly $94%$ of the time. Recall says that our model succsessfully identified $96%$ of all likely incidents and only $4%$ of incidents occured without prior warning.

High Accuracy (0,99) and F1-score (0,95) shows that chosen size of sliding windows ($W=30$) successfully captured the "drift" pattern we addedd prior Incident occurrence. The Random Forest was able to notice the difference between the random noise of normal operations and the structural changes that occur before an incident.

We set the alert threshold manualy at 0.2 level because lower value just leads to more false alarm sytuations meanwhile higher threshold increase possibility of missing Incidents. So this value was chosen after several implementations to keep both metrics in the 90s.
